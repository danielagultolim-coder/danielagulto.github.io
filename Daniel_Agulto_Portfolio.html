FEATURED CASE STUDY

Dan Store — POS App

Live web application, built solo for a real retail business.

THE PROBLEM

Off-the-shelf POS software meant recurring fees for features a single small store didn't need. I built a lightweight system shaped around exactly how the shop runs day to day, at no recurring cost.

THE USERS

Store staff with no technical background, ringing up a sale or checking a customer's balance in seconds — often with the customer standing right there.

THE FLOW

Each tab mirrors one step of an actual shift, in the order staff move through it:

POS→ INVENTORY→ SALES→ UNPAID→ BRANCH

WHY THIS LAYOUT

Staff shouldn't have to think about where something lives. The tab order follows the physical sequence of serving a customer, so the interface stays out of the way instead of adding a decision.
